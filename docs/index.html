What it does

Built for engineers watching Copilot spend.

The interface stays dense and direct. It shows the reports, the forecast, and the audit trail without turning the page into a marketing template.

Raw-first storage

Stores the source payload before parsing so schema changes do not erase history.

  • Raw JSON preserved intact
  • Parsing stays separate from history

Forecasts

Summarizes Copilot usage patterns into simple burn projections and month-end risk.

Budget alerts

Tracks thresholds and notifications without mutating GitHub budget settings.

Dual databases

PostgreSQL in production, SQLite for local dev. Automatically selected based on the scheme of your DATABASE_URL connection string.

Copilot Agent Skills

Trigger forecasts, user classifications, and budget sync checks directly using natural language in your chat interface.

  • On-demand chat commands
  • Seamless CLI script triggers

Quick start

Install, configure, ingest.

Everything below comes straight from the repository workflow and uses the same commands the CLI expects.

Install

$ git clone https://github.com/mhenke/BurnRate.git
$ cd BurnRate
$ npm install

Configure

DATABASE_URL=postgresql://user:pass@localhost:5432/burnrate
GITHUB_TOKEN=ghp_xxx
GITHUB_ENTERPRISE=my-company
GITHUB_ORG=my-org

Create a GitHub Personal Access Token (classic) with read:org permissions to fetch Copilot reports.

Run

# Run migrations to set up database tables
$ npm run migrate
# Ingest Copilot usage reports from GitHub API
$ npm run ingest
# Generate burn forecasts and budget projections
$ npm run forecast

Architecture

Traceable from GitHub API to forecast.

The pipeline keeps raw history first, then parsing and aggregation follow in separate steps.

  1. 1GitHub APIDaily Copilot usage reports
  2. 2ETLRaw storage, parsing, normalization
  3. 3DatabasePostgreSQL or SQLite tables
  4. 4ForecastUsage trend and month-end burn
  5. 5AlertsBudget threshold notifications

Observe-Only & Local-First Security

BurnRate is designed with zero-trust in mind. Your GITHUB_TOKEN and Copilot usage data are processed locally within your own environment and stored in your self-hosted database. BurnRate never transmits your tokens or raw payload logs to any external server, and it performs no write operations back to GitHub.
